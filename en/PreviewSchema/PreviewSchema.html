Schema Metadata

The name, description, and classification is given in the current selected language for the schema. This information can help you decide if you have selected the right schema for your data.

Schema Details

Read the information about each attribute as described in the schema.

Sensitive

Sensitive attributes indicate sensitive data (e.g., names of people, government identification, birth date, etc.).

Unit

Units for each attribute. Not all attributes need to have units.

Type

  • Text: text, from single-letter entries to fields of unstructured text. Numbers are text if you can't perform mathmatical operations on them, e.g. a phone number is text.
  • Numeric: numbers. If you can't put it into a 'calculator' and directly perform mathematical operations on it, then you have text. For example, the range "1056-1067" is text, the value a10 is text because it has a letter in the name etc.
  • Boolean: a data type where the data only has two possible variables: true or false
  • Binary: a data type that defines a binary code signal
  • DateTime: a data type that defines dates. Common formats include dates (e.g., YYYY-MM-DD), times (e.g., hh:mm:ss), dates and times concatenated (e.g., YYYY-MM-DDThh:mm:ss.sss+zz:zz), and durations (e.g., PnYnMnD)
  • Array [attribute type]: a data type that defines a structure that holds several data items or elements of the same data type

Label and Description

Attribute names can sometimes be very cryptic (e.g. v2) or in another language. The label can be a more readable version of the attribute name. The description describes the data that should appear under each attribute.

List

A list of acceptable entries that the user will be able to select from. Only those values that appear in the list would be verified as valid according to the schema rules.

Character Encoding

Documents the character encoding of the data for each attribute.

Required Entries

Specifies if the attribute is required and cannot be left blank in the dataset.

Format Rules

Specifies the formatting rule that applies to the data for each attribute. Behind each rule is an Regular Expression (RegEx) which a computer interprets in order to determine if the data meets the format rule.

Cardinality

For array datatypes only, it specifies the exact, minimum or maximum (or both minimum and maximum) number of entries allowed in a data record for each attribute.
